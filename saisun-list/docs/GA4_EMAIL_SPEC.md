# GA4 メール送信 運用仕様書

## 概要

GA4（Google Analytics 4）のデータを基に、管理者へ日次アラートと週次レポートをメール送信する自動化システム。

- **対象サービス:** デタウリ.Detauri（BtoB古着卸売EC）
- **実装ファイル:** `GA4Analytics.gs`（データ収集）、`GA4Advice.gs`（メール送信）
- **送信先:** スクリプトプロパティ `ADMIN_OWNER_EMAIL` に設定された管理者メール

---

## トリガー構成

| トリガー | 時刻 | 関数 | 役割 |
|---|---|---|---|
| `cronDaily8` | 毎日 8:00 | `ga4SyncAll()` | GA4 API → スプレッドシート（GA4日別サマリー/流入元/イベント）に書き込み |
| `cronDaily9` | 毎日 9:00 | `ga4advice_cron()` | シートデータを読み取り → 曜日で分岐してメール送信 |

**実行順序が重要:** `ga4SyncAll`（8時）でシートが最新化された後に、`ga4advice_cron`（9時）がシートを読む。

---

## 送信分岐ロジック

```
ga4advice_cron()
  ├─ 月曜 (dow === 1)
  │   └─ ga4advice_weeklySummary_()  ← 毎週必ず送信
  └─ 火〜日
      └─ ga4advice_dailyAlert_()    ← 異常検出時のみ送信
```

**エントリポイント:** `GA4Advice.gs:13` `ga4advice_cron()`

---

## 日次アラート（火〜日）

### 送信条件

異常値（±2.0σ）が1つ以上検出された場合のみ送信。異常なしの場合はスキップ。

### 異常値検出ロジック

**関数:** `ga4advice_extractDailyMetrics_()` (`GA4Advice.gs:99`)

1. GA4日別サマリーシートの全データを読み取る
2. **今日の不完全データを除外**（最終行の日付が今日ならスキップ → 前日を「昨日」とする）
3. 直近7日間の平均（μ）と標準偏差（σ）を算出
4. 昨日の値が `|値 - μ| > 2.0σ` の場合に異常として検出

**検出対象メトリクス（4項目）:**

| メトリクス | シート列 | 異常例 |
|---|---|---|
| ユーザー | B列 | 急増 or 急減 |
| 新規 | C列 | 急増 or 急減 |
| セッション | D列 | 急増 or 急減 |
| PV | E列 | 急増 or 急減 |

エンゲージ率・平均滞在時間は異常検出の対象外（7日平均と前週比の表示のみ）。

### メール内容

**件名:** `【デタウリ】GA4アラート: {異常内容}（{日付}）`
  - 例: `【デタウリ】GA4アラート: ユーザー急減, PV急減（2026-03-11）`

**セクション構成:**
1. ⚠ 異常値検出 — 検出されたメトリクスと値・方向・平均±σ
2. 📊 主要指標 — 昨日の全6メトリクス（7日平均・前週比付き）
3. 🌐 流入元 TOP3（昨日）— GA4 API直接呼び出しで昨日1日分を取得

### データソース

| データ | 取得元 | 関数 |
|---|---|---|
| 主要指標・異常値 | GA4日別サマリーシート（シート読み取り） | `ga4advice_extractDailyMetrics_()` |
| 流入元・ファネル | GA4 Data API（`yesterday`〜`yesterday`） | `ga4advice_fetchYesterdayGA4_()` |

## 週次レポート（月曜）

### 送信条件

毎週月曜に必ず送信（異常の有無にかかわらず）。

### メール内容

**件名:** `【デタウリ】週次レポート + AIアドバイス（{日付}）`
  - AI分析失敗時: `【デタウリ】週次レポート ※AI分析なし（{日付}）`

**セクション構成:**
1. 🤖 AIアドバイス（最優先1アクション）— OpenAI GPT-5-mini による分析
2. 🛒 注文サマリー（直近7日）— 注文件数/金額/AOV/前週比/リピート率
3. 📊 GA4主要指標 — 昨日の値 + 7日平均 + 前週比
4. 🔄 ファネル（昨日）— 5ステップ転換率
5. 🌐 流入元 TOP5（昨日）
6. 📋 内訳（直近7日）— チャネル別・決済方法別
7. ⚠ 異常値検出 — ある場合のみ表示

### データソース

| データ | 取得元 | 関数 |
|---|---|---|
| GA4主要指標・異常値 | GA4日別サマリーシート | `ga4advice_extractDailyMetrics_()` |
| 流入元・ファネル | GA4 Data API | `ga4advice_fetchYesterdayGA4_()` |
| 注文データ | 依頼管理シート | `ga4advice_collectOrderMetrics_()` |
| ニュースレター履歴 | ニュースレターシート | `ga4advice_collectWeeklyOrderMetrics_()` |
| AIアドバイス | OpenAI API (`gpt-5-mini`) | `ga4advice_callAI_()` |

### AI分析プロンプト仕様

**関数:** `ga4advice_buildWeeklyPrompt_()` (`GA4Advice.gs:437`)

- system: ECコンサルタントとして「最優先1アクション」を提案
- user: GA4データ + 注文データ + メルマガ配信履歴を構造化テキストで渡す
- 出力フォーマット: `### 最優先アクション` → `### 理由` → `### 想定インパクト`
- ルール: 抽象アドバイス禁止、1人運営考慮、データ数値引用必須

## GA4 データ収集（前段）

### ga4SyncAll()

**ファイル:** `GA4Analytics.gs:29`

GA4 Data API から過去30日分のデータを取得し、3つのシートに書き込む。

| シート名 | 関数 | 内容 |
|---|---|---|
| GA4日別サマリー | `ga4Daily_()` | 日付別のユーザー/新規/セッション/PV/エンゲージ率/平均滞在 + エリアチャート |
| GA4流入元 | `ga4Source_()` | ソース×メディア×キャンペーン別のユーザー/セッション/PV + パイチャート |
| GA4イベント | `ga4Event_()` | イベント別回数 + コンバージョンファネル（5ステップ）+ 棒グラフ |

**日付範囲:** `ga4Range_()` → 今日から30日前〜今日

**注意:** 終了日が「今日」のため、当日の不完全データが最終行に含まれる。日次アラートの `ga4advice_extractDailyMetrics_()` でこれを除外する処理がある。

### GA4 API 設定

- **プロパティID:** `properties/525734643`（`GA4Analytics.gs:10`）
- **必要なGASサービス:** Google Analytics Data API（GASエディタ → サービス(+) → 追加）

### ファネル定義（5ステップ）

| ステップ | GA4イベント名 | ラベル |
|---|---|---|
| 1 | `view_item_list` | 商品一覧表示 |
| 2 | `view_item` | 商品詳細表示 |
| 3 | `add_to_cart` | カート追加 |
| 4 | `begin_checkout` | 注文開始 |
| 5 | `redirect_to_payment` | 決済ページ遷移 |

## 注文データ集計仕様

**関数:** `ga4advice_collectOrderMetrics_()` (`GA4Advice.gs:259`)

### 読み取りシート

依頼管理シート（依頼管理SS）の以下の列を使用:

| 列 | 定数 | 用途 |
|---|---|---|
| B | `REQUEST_SHEET_COLS.DATETIME` | 注文日時 |
| D | `REQUEST_SHEET_COLS.CONTACT` | メールアドレス |
| K | `REQUEST_SHEET_COLS.TOTAL_COUNT` | 点数 |
| L | `REQUEST_SHEET_COLS.TOTAL_AMOUNT` | 金額 |
| O | `REQUEST_SHEET_COLS.PAYMENT_METHOD` | 決済方法 |
| V | `REQUEST_SHEET_COLS.STATUS` | ステータス |
| AG | `REQUEST_SHEET_COLS.CHANNEL` | チャネル |

### 除外条件

ステータスが `キャンセル` or `返品` の行はスキップ。

### 集計項目

| 項目 | 集計期間 | 算出方法 |
|---|---|---|
| 昨日の注文 | 昨日0:00〜今日0:00 | 件数/金額/点数 |
| 直近7日 | 7日前〜今日0:00 | 件数/金額/点数 |
| 前週 | 14日前〜7日前 | 件数/金額/点数 |
| AOV | 直近7日 | 金額 ÷ 件数 |
| 前週比 | 直近7日 vs 前週 | 件数・金額の変化率(%) |
| チャネル別 | 直近7日 | AG列の値ごとの件数 |
| 決済方法別 | 直近7日 | O列の値ごとの件数 |
| リピート率 | 全期間（完了注文） | 2回以上注文したメールアドレス ÷ ユニーク顧客数 |

## メール共通仕様

### HTML生成

`buildHtmlEmail_(opts)` (`Util.gs:340`) を使用。全メールテンプレート（発送通知、入金確認、ニュースレター等）と同じHTML構造。

**opts パラメータ:**
- `greeting`: 挨拶文（日次アラートでは `GA4アラート（日付）`）
- `lead`: リード文
- `sections[]`: セクション配列
  - `title`: セクション見出し
  - `rows[]`: `{ label, value }` のキーバリュー行
  - `items[]`: 箇条書き文字列
  - `text`: 自由テキスト（AI出力等）

### 送信

`ga4advice_sendEmail_()` (`GA4Advice.gs:833`) → `MailApp.sendEmail()` で送信。

- `htmlBody`: HTML版（リッチフォーマット）
- `body`: テキスト版（フォールバック）
- `noReply: true`

## 既知の設計判断

1. **日次アラートにAI分析なし:** コスト・速度の観点から日次はルールベース異常検出のみ
2. **σ計算は母集団分散:** `variance / n`（標本分散 `n-1` ではない）。7日間の小サンプルでは差は微小
3. **今日の不完全データ除外:** `ga4SyncAll` が朝8時に実行され、GA4 API の日付範囲に「今日」を含むため、最終行に当日の不完全データが入る。`ga4advice_extractDailyMetrics_` で最終行の日付が今日の場合はスキップし、前日データで判定する
4. **リピート率は全期間:** 直近7日ではなく完了注文全体での算出（長期指標として）
5. **ファネル最終ステップは `redirect_to_payment`:** `purchase` ではなくKOMOJU決済ページ遷移を最終ステップとしている（購入完了はKOMOJU Webhook経由のため GA4 イベントとして計測されにくい）
