## 注文が入ったら

### 依頼管理シートの見方

注文は「依頼管理」シートに1行1注文で記録される。よく使う列は以下の通り。

| 列 | 名称 | 内容 |
|---|---|---|
| A | 受付番号 | 注文の固有番号（例: R-20260228-001） |
| B | 依頼日時 | 注文が入った日時 |
| C | 会社名/氏名 | お客様の名前 |
| H | 商品名 | 注文された商品（カンマ区切り） |
| I | 確認リンク | 商品確認ファイルのURL（展開後に入る） |
| K | 合計点数 | 注文された商品の点数 |
| L | 合計金額 | 税込の合計金額 |
| O | 決済方法 | クレジットカード / コンビニ払い / 銀行振込 等 |
| Q | 入金確認 | 空=未確認 / 入金待ち / 完了 |
| **S** | **発送ステータス** | **未送付 / 発送済み** ← スタッフが変更する |
| **T** | **配送業者** | **クロネコヤマト等** ← スタッフが入力する |
| **U** | **伝票番号** | **追跡番号** ← スタッフが入力する |
| V | ステータス | 入金待ち / 完了 / キャンセル / 返品 |
| **W** | **担当者** | **自分の名前を入力** |
| **AD** | **備考** | **メモ欄（自由記述）** |

> **太字の列**がスタッフが入力・変更する列。他の列は自動入力されるので触らない。

### 新しい注文の見つけ方

1. V列（ステータス）が空またはオープンの行を探す
2. I列（確認リンク）が入っていれば展開済み → 発送準備に進める
3. I列が空なら未展開 → 依頼展開が必要（自動展開が5分ごとに動くので、通常は待つだけでOK）

## 依頼展開の手順

依頼展開とは、注文データから商品確認ファイルを生成し、仕入れ管理に売却記録をつける作業。

### 自動展開（通常はこれでOK）

5分ごとに自動展開が動いているので、基本的には**何もしなくてよい**。I列に確認リンクが入るのを待つ。

### 手動展開（急ぎの場合）

1. スプレッドシート上部の **「管理メニュー」** をクリック
2. **「1. 依頼展開（展開→XLSX→売却 一括処理）」** を選択
3. ダイアログに **受付番号** を入力して「OK」
4. 処理が完了すると I列に確認リンクが入る

## 欠品時の対応

商品を用意しようとしたら在庫がなかった場合の手順。

1. スプレッドシート上部の **「管理メニュー」** をクリック
2. **「2. 欠品処理（返品→再生成 一括処理）」** を選択
3. ダイアログに**欠品商品の管理番号**を入力（複数ある場合はカンマ区切り）
4. 「OK」を押すと以下が自動で行われる:
   - 欠品商品が仕入れ管理で廃棄済みに変更される
   - 残りの商品で確認ファイルが再生成される
   - お客様への通知メールが送られる（金額も再計算）

> **注意:** 欠品処理は管理者に確認してから実行すること。

## 発送作業

### 発送前の確認

- Q列（入金確認）が「完了」または入金済みであることを確認
  - クレジットカード: 即時に入金確認される
  - コンビニ払い・銀行振込: お客様が支払い後に入金確認が更新される
- **入金が確認できていない注文は発送しない**

### 発送手順

1. 商品を梱包し、配送業者に引き渡す
2. 依頼管理シートで該当注文の行を見つける
3. 以下の列を入力する:

| 列 | 入力内容 |
|---|---|
| T（配送業者） | 配送業者名（例: クロネコヤマト） |
| U（伝票番号） | 配送伝票の追跡番号 |
| **S（発送ステータス）** | **「発送済み」と入力** |

4. S列を「発送済み」に変更すると、**以下が自動で実行される**:
   - お客様に**発送通知メール**が送信される（追跡番号付き）
   - 管理者にも通知メールが送信される
   - V列（ステータス）が自動的に「完了」になる

> **重要:** S列は必ず **「発送済み」** と正確に入力すること。スペースや全角半角の違いがあると自動通知が動かない。

### 発送通知メールの内容

自動送信される発送通知メールには以下が含まれる:
- 商品確認ファイルへのリンク（I列のURL）
- 配送業者名と追跡番号
- お問い合わせ先情報

## 入金待ちの注文の確認方法

### コンビニ払い・銀行振込の注文

1. V列（ステータス）が「入金待ち」の行を探す
2. Q列（入金確認）を確認:
   - 空 or 「入金待ち」→ まだ支払われていない
   - 「完了」→ 入金済み。発送準備に進める
3. 決済期限は注文から**3日間**。期限を過ぎると自動キャンセルされる

### 入金催促

- 毎朝9時に、入金待ちのお客様に自動で催促メールが送信される
- 手動で催促する必要はない

## やってはいけないこと

### 絶対にやらないこと

1. **列（カラム）を削除・挿入しない**
   - 列の追加や削除をすると、すべての自動処理が壊れる
   - 列の順番はシステムで固定されている

2. **行（ヘッダー = 1行目）を編集しない**
   - ヘッダー行はシステムが参照しているので変更しない

3. **A〜R列、X〜AC列、AF〜AG列を手動で書き換えない**
   - これらは自動入力される列。手動で変更するとデータ不整合が起きる
   - 特に P列（決済ID）や Q列（入金確認）は絶対に触らない

4. **シートの名前を変更しない**
   - 「依頼管理」「顧客管理」等のシート名はシステムで使われている

5. **行を並べ替え（ソート）しない**
   - フィルタ表示は使ってもOK、ただし本体のソートは禁止

### 注意が必要な操作

- **V列（ステータス）の変更は慎重に**
  - 「キャンセル」に変更すると元に戻すのが難しい
  - 管理者に相談してから変更すること

- **S列（発送ステータス）は「発送済み」以外を入力しない**
  - 自動通知のトリガーになっているため、誤った値を入れるとメールが送られない

### 困ったときは

- 間違えて値を入力してしまった → **すぐに管理者に連絡**
- 分からない注文がある → AD列（備考）にメモを残して管理者に確認
- シートの動きがおかしい → リロード（ブラウザの更新ボタン）を試す。それでも直らなければ管理者に連絡
